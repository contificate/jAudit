jAudit

Frequently Asked Questions

What platforms does jAudit support?
Since jAudit is written in Java, it can run on any system that supports the JVM. Practically, this means Windows, Mac, and Linux can all run jAudit.
Can jAudit fully decompose AST nodes?
Currently, no. jAudit currently only works upon an overview of the AST. Thus, jAudit cannot infer specific contextual semantics relating to node decomposition. For example, jAudit doesn't differentiate between the two 
BlockStmt
s for the two blocks of an if-else statement.
Can I export the AST depth visualisation?
Currently, no. However, such a feature could definitely be considered as it would be useful to include an SVG version in each audit report.
Can I change the style of the outputted audit report?
Yes, you can simply edit the internal CSS within the outputted HTML files. In future, it could be possible to select different themes from within jAudit and allow multiple audits outputted to the same folder to rely on one global stylesheet.
How can I use a persistent database file?
In order to stop jAudit prompting for a database upon startup, one can pass an absolute path as an argument to jAudit's .jar file. For example: 
java -jar jAudit.jar /path/to/jaudit.sqlite
Does jAudit support languages other than English?
jAudit currently only supports English (en_GB) and German. An open source localisation effort could be possible in future though.
How much does jAudit cost?
jAudit is free and open-source software. If you paid for jAudit, demand a refund.
